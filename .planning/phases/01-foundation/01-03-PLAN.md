**Step 1 — viewport meta에 interactive-widget 추가 (D-15, Android Chrome):**
현재:

변경 후:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover, interactive-widget=resizes-content">
```
`interactive-widget=resizes-content` 추가로 Android Chrome에서 키보드 오픈 시 dvh가 재계산된다.

**Step 2 — syncViewportHeight() 함수 추가 + 초기 실행 (D-17 --vh 폴백, VIEW-02):**
`initKeyboardHandler()` 함수 위(또는 JS Utilities 섹션)에 추가한다:
```javascript
// ═══ SECTION: Viewport Handler ═══
function syncViewportHeight() {
  const vh = (window.visualViewport?.height ?? window.innerHeight) * 0.01;
  document.documentElement.style.setProperty('--vh', `${vh}px`);
}
if (window.visualViewport) {
  window.visualViewport.addEventListener('resize', syncViewportHeight);
  window.visualViewport.addEventListener('scroll', syncViewportHeight);
}
syncViewportHeight(); // 초기 실행 (DOMContentLoaded 이전에 호출되므로 <script> 태그 내 직접 실행)
```
이 코드는 `DOMContentLoaded` 핸들러 밖에 배치해도 되고, `initKeyboardHandler()` 내부에 통합해도 된다. 중요한 것은 `syncViewportHeight()`가 페이지 로드 시 즉시 실행되어야 한다는 것.

CSS에 `--vh` 폴백이 없으면 아래도 추가한다 (Plan 01 SUMMARY 확인 후):
```css
/* SECTION: Reset & Base */
body {
  min-height: 100vh;
  min-height: 100dvh;
  /* --vh 폴백: JS가 --vh를 주입하면 이 calc()가 추가로 적용됨 */
}
.full-height {
  height: calc(var(--vh, 1vh) * 100);
}
```
`.full-height` 클래스는 현재 사용되지 않으면 추가만 해두면 됨 (Phase 2에서 활용).

한글 IME 조합 중 Enter 키로 인한 중복 제출을 방지하는 `isComposing` 가드를 적용한다.

**배경:** Android Chrome에서 한글 입력 시 keydown 이벤트의 keyCode가 229로 오고 `isComposing`이 true다. 이 상태에서 Enter를 누르면 폼이 제출되거나 행이 추가되는 문제가 발생한다.

**Step 1 — 전역 keydown 가드 함수 추가:**
JS Utilities 섹션에 추가한다:
```javascript
// IME 조합 중인지 확인 (한글 입력 가드)
function isComposingEvent(e) {
  return e.isComposing || e.keyCode === 229;
}
```

**Step 2 — 폼 submit keydown 가드:**
`handleSubmit()` 함수가 Enter 키로 트리거되는 경로를 확인한다.
현재 `<form id="entryForm" onsubmit="handleSubmit(event)">` 형태라면 폼 submit 이벤트에는 isComposing이 이미 필터됨 (submit 이벤트는 IME 완료 후 발생). 그러나 텍스트 input에서 Enter 시 submit이 트리거되는 경우 처리 필요.

방법: DOMContentLoaded 이벤트 핸들러 안에서 폼의 텍스트 input들에 keydown 리스너를 추가한다:
```javascript
// ═══ 한글 IME 가드: Enter 키로 폼 제출 방지 ═══
document.querySelectorAll('#entryForm input[type="text"], #entryForm input[type="number"]').forEach(inp => {
  inp.addEventListener('keydown', e => {
    if (e.key === 'Enter' && isComposingEvent(e)) {
      e.preventDefault();
      e.stopPropagation();
    }
  });
});
```
이 코드를 `DOMContentLoaded` 핸들러 안 `addRow()` 호출 이후에 배치한다.

**Step 3 — renderRows()로 동적 생성되는 input에도 가드 적용:**
renderRows()로 생성되는 `<input>` 태그들은 이벤트 위임으로 처리한다. `document.getElementById('taskTableBody')` 에 이벤트 위임 리스너를 추가한다:
```javascript
// 동적 생성 input에 대한 IME 가드 (이벤트 위임)
document.getElementById('taskTableBody').addEventListener('keydown', e => {
  if (e.target.tagName === 'INPUT' && e.key === 'Enter' && isComposingEvent(e)) {
    e.preventDefault();
    e.stopPropagation();
  }
});
```
이 리스너는 `DOMContentLoaded` 핸들러 안에 배치한다 (taskTableBody가 로드된 후).

**Step 4 — PIN 입력 keydown 가드 확인:**
기존 PIN keydown 핸들러 (약 504줄)에는 `Backspace` 처리만 있어 IME 가드 불필요. 건드리지 않는다.

**주의:**
- `e.preventDefault()`는 Enter 키의 폼 제출 동작을 막는 것이지, 한글 문자 입력 자체를 막는 게 아님.
- isComposing은 Chrome(Android), Safari(iOS) 모두 지원. keyCode 229는 일부 구형 Android 브라우저를 위한 폴백.
- `oninput` 이벤트로 updateRow()를 호출하는 경우(현재 코드)는 IME 완료 후에만 발생하므로 가드 불필요.
