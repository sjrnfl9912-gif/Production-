**브라우저 DevTools (Chrome)에서 확인:**

1. index.html을 브라우저에서 열기 (또는 로컬 서버 사용)

2. **UI-01 — 실적 입력 카드 레이아웃:**
   - DevTools → 반응형 모드 → 375px 설정
   - "실적 입력" 탭 활성화
   - task-table 행이 카드 형태로 표시되는지 확인 (작업유형, 작업명, 수량 각 줄 표시)
   - 가로 스크롤 없이 화면 내에 표시되는지 확인

3. **UI-02 — 대시보드 KPI/테이블 모바일 최적화:**
   - "주간 대시보드" 탭 클릭
   - 375px: KPI 카드가 2열(행당 2개)로 표시되는지 확인
   - 375px: 대시보드 테이블 컬럼이 내용에 맞게 자동 크기 조정되는지 확인
   - DevTools → 1280px 변경: KPI 카드가 3열로 표시되는지 확인 (회귀 없음)

4. **UI-03 — 탭바 액티브 인디케이터:**
   - 375px 모바일 뷰
   - 활성 탭 버튼 상단에 파란색 3px 가로선이 보이는지 확인
   - 탭을 클릭해서 전환 시 인디케이터가 이동하는지 확인
   - 탭바가 화면 하단에 홈 바 위로 표시되는지 확인

5. **UI-04 — 스켈레톤 로딩:**
   - DevTools → Network 탭 → "Slow 3G" 스로틀 선택
   - 페이지 새로고침 (Ctrl+Shift+R)
   - KPI 영역, 주간 현황 영역에 shimmer 애니메이션 스켈레톤이 1-3초간 표시되는지 확인
   - 로딩 완료 후 실제 데이터로 교체되는지 확인
   - Network "No throttling" 복원 후 탭 전환 시 스켈레톤 미표시 확인

6. **UI-05 — 내 실적 수정 날짜 그룹화:**
   - "내 실적 수정" 탭 클릭
   - 작업자 선택 후 기록이 날짜별로 그룹화되어 날짜 헤더가 보이는지 확인
   - 날짜 헤더에 "기한 만료" 레이블이 7일 초과 날짜에 표시되는지 확인
   - 7일 이내 기록에 수정/삭제 버튼이 표시되는지 확인
   - 작업자 select가 스크롤 시 상단에 고정되는지 확인

7. **회귀 확인:**
   - 실적 입력 및 저장이 정상 동작하는지 확인 (폼 작성 → 저장 버튼)
   - 삭제 시 customConfirm 모달이 표시되는지 확인
   - 1280px 데스크탑: 레이아웃 변화 없는지 확인
