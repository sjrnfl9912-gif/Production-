**변경 2: SECTION: Animations (line 178) 와 SECTION: Desktop Override (line 185) 사이에 새 Touch UX 섹션 삽입**

`/* ═══ SECTION: Animations ═══ */` 블록이 끝나는 줄(line 184, `@keyframes spin{...}`) 직후,
`/* ═══ SECTION: Desktop Override ═══ */` (line 185) 직전에 아래 블록을 삽입:

```css
/* ═══ SECTION: Touch UX ═══ */
/* TOUCH-01: 최소 44x44px 터치 타겟 (터치 장치에서만 적용) */
/* TOUCH-02: touch-action on inputs (버튼류는 이미 적용됨) */
/* TOUCH-03: 시각적 터치 피드백 — scale + opacity + ripple */
@media (pointer: coarse) {
  .btn-icon{width:44px;height:44px}
  .week-nav-btn{width:44px;height:44px}
  .btn-sm{padding:10px 14px;min-height:44px}
  .settings-item .btn{padding:8px 12px;min-height:44px}
  input,select,textarea{touch-action:manipulation}
  .btn:active,.btn-add:active,.btn-icon:active,.week-nav-btn:active{transform:scale(0.94);opacity:0.82;transition:transform 0.05s,opacity 0.05s}
  .tab-btn:active{background:var(--pri-lt);color:var(--pri)}
}
/* TOUCH-03: Ripple — btn-primary / btn-danger CTA 버튼 */
.btn-primary::after,.btn-danger::after{content:'';position:absolute;inset:0;background:rgba(255,255,255,0.25);opacity:0;transform:scale(0);border-radius:inherit;pointer-events:none;transition:transform 0.3s,opacity 0.3s}
.btn-primary:active::after,.btn-danger:active::after{transform:scale(1);opacity:1;transition:none}
```

**변경 3: iOS :active 즉시 발화를 위한 최소 touchstart 리스너 추가 (JS)**

iOS Safari에서는 `document`에 touchstart 리스너가 하나도 없으면 `:active` CSS 상태가 즉시 발화하지 않는다.
02-02 계획의 `initSwipeNav()`도 touchstart를 등록하지만, 02-01은 Wave 1에서 독립 실행되므로 자체적으로 이 리스너를 포함해야 한다.

DOMContentLoaded 핸들러 안 — 탭 클릭 핸들러 직전 또는 직후의 적절한 위치에 아래 한 줄을 삽입한다:

```javascript
// iOS :active 즉시 발화 보장 (TOUCH-03) — passive touchstart 리스너 등록
document.addEventListener('touchstart',function(){},{passive:true});
```

**주의사항:**
- 모든 터치 타겟 크기 변경은 반드시 `@media (pointer: coarse)` 안에 있어야 한다. 데스크탑(pointer:fine)에는 적용되지 않는다.
- `.btn-sm`의 데스크탑 오버라이드(SECTION: Desktop Override의 `.settings-item .btn{padding:5px 12px}`)는 `min-width:769px`이므로 pointer:coarse 블록과 충돌하지 않는다. 데스크탑은 pointer:fine이라 coarse 블록이 무시된다.
- ripple `.btn-primary::after`는 `@media` 밖에 위치시킨다 — 데스크탑 hover에서도 ripple 효과는 무해하다.
- JS 리스너는 빈 핸들러(`function(){}`)로 충분 — 단지 등록 자체가 목적. `passive:true`는 필수.
- 02-02가 실행되면 `initSwipeNav()`의 touchstart 리스너가 이 역할을 겸한다. 중복 등록은 무해하다.
