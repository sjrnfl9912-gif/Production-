**변경 1: .overflow-x 선언 수정 (line 169)**

현재:
```css
.overflow-x{overflow-x:auto;-webkit-overflow-scrolling:touch}
```

변경 후:
```css
.overflow-x{overflow-x:auto;-webkit-overflow-scrolling:touch;scroll-snap-type:x proximity;overscroll-behavior-x:contain}
```

이유:
- `scroll-snap-type:x proximity` — proximity는 근처에서만 스냅, mandatory는 모든 스크롤 후 강제 스냅하여 사용 불편 (RESEARCH.md Pitfall 3)
- `overscroll-behavior-x:contain` — 수평 스크롤이 페이지 body로 전파되지 않도록

**변경 2: SECTION: Dashboard에 .dash-table th 스냅 정렬 추가**

SECTION: Dashboard 내 `.dash-table th{...}` 선언을 찾아 `scroll-snap-align:start`를 추가:

현재:
```css
.dash-table th{padding:8px 10px;background:#f4f5f7;font-weight:600;...;white-space:nowrap}
```

변경 후 — 기존 선언 끝에 `scroll-snap-align:start` 추가:
```css
.dash-table th{padding:8px 10px;background:#f4f5f7;font-weight:600;...;white-space:nowrap;scroll-snap-align:start}
```

이유: 테이블 헤더 열마다 스냅 포인트를 설정하여 컬럼 단위로 스냅함. td가 아닌 th에만 적용하여 스냅 밀도를 줄임 (매 셀이 아닌 헤더 기준).

탭 클릭 핸들러 (line 514, `});` 닫는 줄) 바로 다음에 아래 JS 블록을 삽입한다.

```javascript
// ═══════════ SWIPE TAB NAVIGATION (TOUCH-04) ═══════════
(function initSwipeNav(){
  const TAB_ORDER=['input','myrecords','dashboard','admin'];
  const MIN_X=60;  // px: 스와이프로 인정하는 최소 수평 이동
  const MAX_Y=80;  // px: 이 이상 수직 이동이면 스크롤로 간주, 스와이프 취소
  let sx=0,sy=0;

  // passive:true — scroll을 절대 막지 않음 (RESEARCH.md Pitfall 1)
  // 이 touchstart 리스너는 iOS :active 즉시 발화도 보장함 (RESEARCH.md Pitfall 2)
  document.addEventListener('touchstart',function(e){
    sx=e.touches[0].clientX;
    sy=e.touches[0].clientY;
  },{passive:true});

  document.addEventListener('touchend',function(e){
    // PIN 입력 중 스와이프 무시 (RESEARCH.md Pitfall 5)
    if(document.activeElement&&document.activeElement.classList.contains('pin-digit'))return;
    const dx=e.changedTouches[0].clientX-sx;
    const dy=e.changedTouches[0].clientY-sy;
    if(Math.abs(dx)<MIN_X)return;    // 수평 이동 부족
    if(Math.abs(dy)>MAX_Y)return;    // 수직 스크롤로 판단
    const dir=dx<0?1:-1;             // 왼쪽 스와이프=다음 탭, 오른쪽=이전 탭
    const cur=document.querySelector('.tab-btn.active')?.dataset.tab;
    const ci=TAB_ORDER.indexOf(cur);
    const ni=ci+dir;
    if(ni<0||ni>=TAB_ORDER.length)return;
    // 기존 탭 클릭 핸들러 재사용 — 새 탭 전환 로직 없음
    document.querySelector(`.tab-btn[data-tab="${TAB_ORDER[ni]}"]`)?.click();
  },{passive:true});
})();
```

삽입 위치: `// ═══════════ PIN SYSTEM ═══════════` 주석 직전 (line 516 근처).

**주의사항:**
- `{ passive: true }` 는 두 리스너 모두에 필수. 없으면 Chrome이 경고하고 scroll 성능이 저하됨.
- `preventDefault()` 는 절대 호출하지 않는다.
- `TAB_ORDER`는 하드코딩 — DOM 순서와 일치하므로 안전. 탭 추가 시 양쪽 모두 업데이트 필요.
- `.pin-digit` 체크는 `classList.contains` 사용 — `tagName`이 아닌 클래스 기준.
